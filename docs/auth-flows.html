Maple Diary - Authentication Flows

Admin: password login + OTP-verified sign-up | Teacher & Student: OTP-verified login via ConvexCredentials

Admin Portal Email + Password
Sign In (no OTP needed)
1
Enter credentials CLIENT
email password
2
Verify password SERVER
ConvexCredentials authorize() looks up admin by email, verifies bcrypt hash
3
Session created CLIENT
Redirect to /dashboard
-- or --
Create Account (OTP required)
1
Enter details CLIENT
name email password confirm
2
Check email available SERVER
requestAdminSignUpOtp() verifies email not taken
3
Send OTP EMAIL
6-digit code sent to email (5 min expiry)
4
Enter OTP CLIENT
User enters 6-digit code from email
5
Verify OTP + Create SERVER
completeAdminSignUp() verifies OTP, hashes password (bcrypt), creates admin user
6
Auto sign-in CLIENT
Calls signIn("admin") with new credentials, redirect to /dashboard
Route Guard
AdminAuthGuard checks useConvexAuth() + verifies role === "admin" via currentUser query. Redirects to /sign-in if unauthorized.
Teacher Portal Teacher ID + Email/Phone + OTP
Step 1 -- Credentials
1
Enter credentials CLIENT
teacherId email or phone
2
Validate credentials SERVER
requestTeacherOtp() looks up staff by email/phone, verifies teacherId match + isActive
3
Send OTP EMAIL
6-digit code sent via Resend to staff email (5 min expiry)
Step 2 -- OTP Verification
4
Enter OTP code CLIENT
User enters 6-digit code from email
5
Verify OTP + Credentials SERVER
ConvexCredentials authorize() verifies OTP, then re-validates teacherId + email/phone against staff table. Creates auth user if first login.
6
Session created CLIENT
Session token stored in localStorage (maple-teacher). Redirect to /teacher/dashboard
Route Guard
TeacherAuthProvider checks useConvexAuth() + verifies role === "teacher" via currentUser query. Fetches staff profile. Redirects to /teacher/login if unauthorized.

OTP Security Rules
Server-Side Enforcement
  • Teacher/Student: OTP verified inside ConvexCredentials authorize() -- cannot bypass
  • Admin sign-up: OTP verified in completeAdminSignUp() action before account creation
  • Admin sign-in: No OTP needed (email + password only)
  • 6-digit numeric code, cryptographically random
  • 5 minute expiry from generation
  • Max 5 failed attempts per OTP code (then locked)
  • Single-use: marked as used immediately after verification
  • Previous OTPs invalidated when new one is requested
  • 60-second resend cooldown on client
Route Protection
Defense Layers
  • Client: useConvexAuth() checks isAuthenticated before rendering
  • Client: Role verification via currentUser query (admin/teacher/student)
  • Client: Loading spinner while auth resolves (no content flash)
  • Server: getAuthUserId(ctx) validates every Convex query/mutation
  • Isolation: Separate ConvexReactClient per portal with namespaced storage
  • Login pages redirect away if already authenticated
  • Admin passwords: bcrypt hashed (10 salt rounds)
Account Recovery
Recovery Flows
  • Admin: resetAdminPassword internal action (Convex dashboard)
  • Teacher: Forgot Password -- reset link via email (1hr expiry)
  • Teacher: Reset token: 48-char random, single-use, clears on success
  • Student: Forgot Student ID -- ID reminder sent to email
  • All recovery responses don't reveal if account exists
  • Password minimum: 6 characters
